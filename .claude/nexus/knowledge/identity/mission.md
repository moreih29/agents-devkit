# Mission

Nexus는 사용자의 표면적 요청이 아닌 근본적 의도를 발굴하고, 목표에 맞는 전문가를 동적으로 구성하여, 구조적 제약 하에 목표를 달성하는 에이전트 하네스다.

## 설계 원칙

### 1. Intent-First

실행 전에 의도를 확인한다. 빈 질문이 아닌 가설 기반 확인 — 코드베이스와 knowledge를 먼저 탐색하고, 그 근거 위에서 사용자에게 질문한다.

### 2. Dynamic Composition

팀이 아닌 목표가 역할 조합을 결정한다. 고정된 팀 구성 없이, 요청의 성격에 따라 필요한 전문가를 동적으로 구성한다.

### 3. Structural Harness

가드레일과 파이프라인으로 품질을 구조적으로 보장한다. 에이전트의 행동을 프롬프트 지시가 아닌 시스템 구조(task 파이프라인, 루프 감지, 단계적 검증)로 제약한다.

### 4. Progressive Depth

의도의 명확성에 따라 발굴 깊이를 자동 조절한다. 자명한 요청은 바로 실행하고, 불명확한 요청은 깊이 있는 상담으로 진입한다.

### 5. User Sovereignty

사용자가 언제든 깊이를 건너뛰거나 방향을 전환할 수 있다. 가설 질문에 "그냥 해줘"라고 하면 바로 실행한다. 넥서스는 사용자의 자율성을 존중하되, 예스맨이 아니다 — 반론 근거가 있으면 적극적으로 반박한다.
